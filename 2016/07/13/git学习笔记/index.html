git学习笔记

2016-07-13 | 分类： git | 标签： git

本文参考自：Git教程 - 廖雪峰的官方网站、git简易指南

git的介绍

进行过团队编程的都是用过版本管理工具，比如我之前学习j2ee时就会和同学组成团队进行编程
完成web项目。当时使用的是svn感觉还是挺好用的。后来在网上搜索”版本工具时发现svn不是最好
的版本管理工具（个人觉得挺好用的呀），不管了既然git更好的话，我就了解下。和网上介绍一样
git是分布式的，而svn是集中式的。具体的介绍还是百度吧。

git的工作流

你的本地仓库（repository）由git维护的三棵“树”组成。第一个是你的工作目录，它持有实际文件；第二个是缓存区（Index），它像个缓存区域，临时保存你的改动；最后是HEAD(即本地代码仓库)，指向你最近一次提交后的结果。
同时你的代码也会有一个远程仓库，你可以将本地代码仓库中的代码合并到远程仓库。

git的命令

1、首先需要“自报家门”

$ git config --global user.name "Your Name"
$ git config --global user.email "email@example.com"

2、建立本地仓库

git init

3、将项目导入仓库中（直接新建项目、拉取远程仓库项目）

git status//查看当前文件状态，看修改了那些文件，那些文件要放入缓存区
git diff <filename>//查看filename文件做了什么修改（对比本地参库）
git add <filename>//添加filename到缓存区
git commit -m “介绍本次提交代码内容” //将缓存区代码提交到本地仓库

4、代码回退

在编码时难免会遇到做了错误的修改，这时你就急需一颗“后悔药”。

git log//查看代码提交历史记录（这时你就知道问什么提交时要写说明了吧，不然鬼知道你那时提交了啥）
git log --pretty=oneline//精简版log
git log --graph//查看分支合并情况
git reset <commit_id>//代码回退到commit_id提交时的状态  
git reflog//查看命令历史记录

5、管理修改

git管理的是修改而不是文件，就是说如果你修改了将文件放入缓存区的是修改，提交的也是修改。所以你在将
文件修改放入缓存区后做的修改都不会被提交。

6、撤销修改

git checkout -- <filename>
命令git checkout -- readme.txt意思就是，把readme.txt文件在工作区的修改全部撤销，这里有两种情况：
一种是readme.txt自修改后还没有被放到暂存区，现在，撤销修改就回到和版本库一模一样的状态；
一种是readme.txt已经添加到暂存区后，又作了修改，现在，撤销修改就回到添加到暂存区后的状态。
git reset HEAD <filename>

7、删除文件

git rm <filename>//git status发现本地文件删除了，确实需要从仓库中删除则使用此命令
git checkout -- <filename> //是误删，则可以轻松找回

8、远程仓库

github是一个提供git远程仓库服务的网站，不过也有其它网站比如“coding”、“开源中国”，也可以自己搭建git服务器
话说我也没搭建过想了解的可以找度娘，这里主要介绍下使用github做为远程仓库。
本地仓库与github上的远程仓库之间进行文件传输是使用了SSH_KEY进行的。
1、首先要有一个github账号
2、配置SSH_KEY百度经验 window下配置SSH连接GitHub、GitHub配置ssh key
3、将本地仓库关联至远程仓库、并推送代码

$ git remote add origin git@github.com:michaelliao/learngit.git//关联远程仓库
$ git push origin master//推送本地master分支更改到远程origin库
$ git clone git@github.com:michaelliao/gitskills.git//克隆版本到本地
$ git remote//查看远程库信息
$ git remote -v//查看详细远程库信息

9、分支的使用

查看分支：git branch
创建分支：git branch <name>
切换分支：git checkout <name>
创建+切换分支：git checkout -b <name>
合并某分支到当前分支：git merge <name>
删除分支：git branch -d <name>
合并分支时如果两个分支中的修改有冲突需要先解决冲突才能完成合并
$ git merge --no-ff -m "merge with no-ff" dev//默认合并分支是fast forward模式，在删除分支后会丢失分支信息，--no-ff是用于禁用fast forward，这样就能在分支历史中查看到分支信息了
git branch -D <name>//强行删除未合并分支（丢失修改）

10、暂停存档

好像好厉害的样子，就是你在a分支上工作但是没有完成（不能提交），但这时你要到另外一个b分支去解决一个bug，这时你就需要将当前在a分支上的修改记录进行存档，然后再切换分支。

git stash//存档
git stash list//查看存档历史
git stash apply stash@{0}//读取存档
git stash pop stash@{0}//读取存档并删除存档

11、标签

git tag v1.0//当前分支head添加tag
git tag//查看tag
git tag v0.9 <commit_id>//在commit_id处添加tag
git show v0.9显示v0.9标签信息
git tag -a "v0.1" -m "version 0.1 released" <commit_id>//添加tag并添加说明
PGP签名标签
git tag -d v0.1//删除本地v0.1tag
git push origin --tags//推送所有本地未提交的tag
git push origin :refs/tags/<tagname>//删除远程tag

12、其他配置

1、配置颜色

$ git config --global color.ui true

2、配置别名

$ git config --global alias.st status
$ git config --global alias.co checkout
$ git config --global alias.ci commit
$ git config --global alias.br branch
$ git config --global alias.lg "log --color --graph --pretty=format:'%Cred%h%Creset -%C(yellow)%d%Creset %s %Cgreen(%cr) %C(bold blue)<%an>%Creset' --abbrev-commit"

3、配置文件

  • 每个仓库的Git配置文件都放在.git/config文件中
  • .gitignore文件，配置git忽略文件
  • 当前用户的Git配置文件放在用户主目录下的一个隐藏文件.gitconfig中
