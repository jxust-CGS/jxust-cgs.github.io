内存溢出、栈溢出与热部署

2018-02-01 | 分类： java | 标签： java

java中的堆heap与栈stack

JVM调优总结（一）– 一些概念 - 和你在一起 - ITeye博客
堆中存的是对象。
栈中存的是基本数据类型和堆中对象的引用。
一个对象的大小是不可估计的，或者说是可以动态变化的，但是在栈中，一个对象只对应了一个4btye的引用（堆栈分离的好处：））。

内存溢出(OutOfMemoryError)

我们这里提到的内存溢出其实就是堆空间不足

首先、通过示例查看内存溢出原因是什么？
java内存溢出示例(堆溢出、栈溢出) - THISISPAN - 博客园


堆内存分为3块年轻代（young generation)、年老代（old generation）、持久代（permanent generation） ，其中年轻代分为eden区以及2个（或多个）Survivor区

jvm中配置堆内存参数：

  • -Xms 是指设定程序启动时占用堆内存大小。一般来讲，大点，程序会启动的快一点，但是也可能会导致机器暂时间变慢。
  • -Xmx 是指设定程序运行期间最大可占用的堆内存大小。如果程序运行需要占用更多的内存，超出了这个设置值，就会抛出OutOfMemory异常。

堆空间不足(OutOfMemoryError: Java heap space)

  • 内存溢出是由于java程序引用对象过多超出了jvm的空间大小
    • 优化程序避免不必要的对象引用（如出现循环创建对象）
    • 调整vm参数,增大-Xmx最大堆空间

永久代溢出(OutOfMemoryError: PermGen space)

  • 程序中声明了过多的静态变量及方法
    • 设置-XX:MaxPermSize增大空间

栈溢出(StackOverflowError)

栈溢出是由于java代码的循环调用、或方法调用层次超出jvm的堆栈空间（如执行递归算法）
-Xss是指设定每个线程的堆栈大小。这个就要依据你的程序，看一个线程大约需要占用多少内存，可能会有多少线程同时运行等。JDK5.0以后每个线程堆栈大小默认为1M

内存泄漏

  • 内存泄漏(memory leak)指由于疏忽或错误造成程序未能释放已经不再使用的内存的情况。内存泄漏并非指内存在物理上的消失，而是应用程序分配某段内存后，由于设计错误，失去了对该段内存的控制，因而造成了内存的浪费。
  • 通过这种有向图的内存管理方式，当一个内存对象失去了所有的引用之后，GC 就可以将其回收。反过来说，如果这个对象还存在引用，那么它将不会被 GC 回收，哪怕是 Java 虚拟机抛出 OutOfMemoryError 。

热部署

使用idea开发设置tomcat热部署项目避免修改后手动编译部署
修改服务器配置，使得IDEA窗口失去焦点时，更新类和资源
菜单Run -> EditConfiguration , 然后配置指定服务器下，右侧server标签下on frame deactivation = Update classes and resource。

基于JVM提供的热加载仅支持方法块内代码修改，只有debug模式下，并且是在idea失去焦点时才会触发热加载，相对加载速度缓慢
