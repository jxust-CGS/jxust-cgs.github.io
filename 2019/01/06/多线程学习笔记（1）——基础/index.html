多线程学习笔记（1）——基础

2019-01-06 | 分类： java | 标签： java

线程的启动

java中启动一个新的线程的方式：调用继承了Thread类的start()方法或实现了Runabled接口的类的start()方法

线程类（Thread）的常用方法

  • isAlive()：判断线程是否在执行
  • sleep()：静态方法，将当前线程暂停一段时间
  • getId()：获取线程的id
  • interrupt():发送中断信息
  • interrupted():静态方法，获取所在线程是否中断并清除中断信号
  • isInterrupted():获取是否中断
  • stop():过时方法，不建议使用，直接停止线程，并强制解除锁导致数据不一致
    停不下来的线程，只能通过在线程中监听中断信息的变化自行处理中断任务

sleep方法中会去检查是否线程状态是否中断，如果为true则抛出异常InterruptedException，从而达到中断线程的目的

suspend 与 resume

虽然与stop方法一样，这两个方法也是过时的，也了解一下
suspend与resume是线程的暂停与继续方法，能够暂停当前线程，但是如果在线程中加入了同步锁synchronized锁，那么当一个线程被暂停那么这个线程的锁将不会释放，如果这时程序中忘记或遗漏掉了resume方法，那么这个线程间一直处于锁定的状态。当有其他线程再次访问加锁的对象将会一直处于等待。
ps：System.out.println方法中有synchronized锁

yield

静态方法，yield切换到其他线程，即让当前运行的线程放弃对cpu的占据，让其他线程运行
但是当cpu再次空闲将继续执行

setPriority

设置当前线程的优先级别，系统默认优先级别1~10，级别不在此范围内将会抛出错误
setPriority具有继承特性、规律性、随机性

  • 继承性：如果A线程中启动B线程，那么B线程的优先级将和A线程保持一致
  • 规律性：优先级较高的线程将会先执行，能够分配到cpu时间更多运行任务执行更快
  • 随机性：优先级更好是会优先抢占cpu时间但是也不是优先级更好的线程一定早于优先级更低的线程执行。个人理解优先级就是类似高概率

setDaemon

守护线程：当非守护线程全部结束，守护线程即自行结束
如果守护线程的存在时间不够长也会早于其他线程结束任务，但是当守护线程的一直处于运行中，这时才能应用到此特性，当其他所有非守护线程结束，守护线程就会自行结束任务

此示例中Main、B、C这3个线程是非守护线程、A是守护线程：原本运行时长最长的A任务在其他非守护线程任务时长最长的C任务之后就直接结束了

synchronized

在线程中为了避免多线程访问同一个对象，调用时数据的脏读取，导致程序异常，我们需要给程序加上同步锁，也就是使用synchronized关键字
synchronized是为当前对象添加或指定的对象同步锁，对于多个对象并行访问多个对象的情况下是不存在脏读取的问题，页面不存在线程不安全一说
只有在多个线程同时访问同一个对象的情况下才需要添加同步锁
synchronized如果是添加在方法上如：
