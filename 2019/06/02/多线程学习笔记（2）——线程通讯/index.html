多线程学习笔记（2）——线程通讯

2019-06-02 | 分类： java | 标签： java

线程之间的通信

生产者消费者模型

join

join方法是等待线程执行完成后才能继续当前线程的执行，等待时间为指定长度

输出结果

按照程序执行逻辑main线程的输出必须在b线程执行完成之后，但是由于设置了1秒的等待时间，而b线程在程序开始时等待a线程释放锁，执行时长大于1秒所以main线程先执行完成

join、wait、sleep比较

  • join让当前线程暂停，等待指定线程（可指定等待时长），释放当前锁
  • wait暂停当前线程，并等待唤醒通知（可指定等待通知时长），释放当前锁
  • sleep暂停线程指定的时长（必须指定时长），不释放当前锁

ThreadLocal

执行结果

ThreadLocal,为每个线程单独创建一个单独的存储空间，存放数据
InheritableThreadLocal，为每个线程单独创建一个存储空间，但是子线程会继承父线程中的值，且在子线程中修改后的值并不会影响父线程的值

PipeReader

通过管道流实现线程间通讯
PipedReader字符流
PipedInputStream字节流
具体使用可以类比java中的文件字符流、字节流
